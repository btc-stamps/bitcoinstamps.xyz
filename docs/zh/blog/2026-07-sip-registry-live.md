# SIP 注册表正式上线

Bitcoin Stamps 改进提案注册表现已在 bitcoinstamps.xyz 发布。它是 <EntityMention entity="bitcoin-stamps">Bitcoin Stamps</EntityMention> 协议如何演变的规范、带版本的所在，并且向所有人公开可读。

## SIP 是什么

SIP 就是 Bitcoin Stamps 改进提案（Bitcoin Stamps Improvement Proposal）。它是针对协议某项改动的书面规范：它做什么、为什么需要、如何编码，以及哪个区块激活它。SIP 是社区把一个想法变成共识规则、且不必进行幕后决策的方式。

白皮书曾抽象地描述了这套治理框架。注册表把它落到了实处。如今每一份提案都有一个编号、一个状态，以及一个你可以指向的页面。

## 已发布的内容

注册表首发即收录 13 份 SIP：

- **SIP-0000**，定义 SIP 如何运作的流程文档，一直贯穿到 **SIP-0011**。
- **SIP-0110**，Ordinals Provenance。

它们合在一起，既覆盖了协议已经在其上运行的各项标准，也覆盖了塑造未来走向的各项提案。每一条都独立成页，因此你能读到确切的规范本身，而不是它的一份摘要。

## 为什么注册表很重要

你无法检视的治理，算不上真正的治理。把这些 SIP 放进一个带版本的统一之处，一次做成了好几件事：

- 建设者获得一个关于编码和激活规则的单一事实来源。
- 新贡献者能看到决策的完整历史，而不仅仅是当前状态。
- 任何人都能核对：索引器所做的，是否与某份 SIP 所规定的一致。

最后这一点把注册表重新系回了协议的核心习惯：对照链去证明它，而不只是断言它。一份已发布的 SIP，加上一个开源的索引器，意味着同一个人就能把一条规则读到、跑起来，并加以验证。
