# Bitcoin Stamps 经济学

了解 Bitcoin Stamps 与其他比特币协议之间的成本差异。

## 存储方式比较

### Bitcoin Stamps：永久 UTXO 存储

**您支付的内容：**
- 比特币 UTXO 集合中的**永久存储**
- **每个比特币节点**永久存储您的数据
- **真正的永久性** - 比特币核心区块链状态的一部分
- **更高成本**但拥有最大安全性和可访问性

### Ordinals：见证数据存储

**您获得的内容：**
- **见证数据存储**（不是比特币核心状态的一部分）
- 由于见证数据享有 75% 折扣，**费用折扣**
- **有限节点存储** - 估计不到 20% 的比特币节点存储见证数据
- **更低成本**但永久性保障降低

## 为什么价格有差异？

**简单解释：**
- **Bitcoin Stamps** = 在每个比特币节点上永久存储数据
- **Ordinals** = 在见证部分存储数据（不保证在所有节点上）
- **结果** = Bitcoin Stamps 成本更高但提供真正的永久性

您支付的是在世界上最安全网络上的**有保障的永久存储**。

### 技术存储差异

**Bitcoin Stamps（UTXO 集合）：**
```typescript
// 数据存储在实际比特币输出中
{
  value: 546,                    // 最小 UTXO 值
  scriptPubKey: "P2WSH 输出",    // 包含邮票数据
  // 存储在 UTXO 集合 = 永久
}
```

**Ordinals（见证数据）：**
```typescript
// 数据存储在见证部分
{
  witness: ["<inscription_data>"], // 获得 75% 费用折扣
  // 不保证在所有节点上
}
```

## 节点存储现实

**Bitcoin Stamps：**
- 存储在**100% 的比特币节点**（UTXO 验证所必需）
- 比特币核心区块链状态的一部分
- 不能在不破坏共识的情况下修剪

**Ordinals：**
- 估计存储在**不到 20% 的比特币节点**
- 见证数据在验证后可以丢弃
- 许多节点以修剪模式运行，没有见证数据

## 文件大小和成本关系

**交易限制：**
- **标准中继**：每笔交易 100KB（比特币核心默认）
- **非标准**：每笔交易最多 1MB（取决于节点）
- **SRC-721 递归**：通过层组合无限

**成本示例（20 sats/vB）：**
- **小邮票（10KB）**：15-30k 聪（$20k BTC 时约 $3-6）
- **大邮票（50KB）**：50-100k 聪（$20k BTC 时约 $10-20）
- **最大单笔交易（100KB）**：100-200k 聪（$20k BTC 时约 $20-40）
- **SRC-20 代币部署**：20-40k 聪（$20k BTC 时约 $4-8）

*成本随比特币网络状况变化*

## 价值主张

**为什么为 Bitcoin Stamps 支付更多？**
- **真正的永久性**：保证在所有比特币节点上存储
- **最高安全性**：受比特币完整共识保护
- **无持续成本**：永久存储一次性付款
- **通用可访问性**：可从任何比特币节点检索
