# Bitcoin Stamps 与 Ordinals/Inscriptions：技术架构

Bitcoin Stamps 和 Ordinals/Inscriptions 代表了在比特币上存储数据的两种根本不同的方法。对于希望确保其数字资产具有最强永久性保证的开发者和用户来说，了解这些差异至关重要。

## 关键技术差异

### 数据存储位置

**Bitcoin Stamps：**
- 使用 **P2WSH（Pay-to-Witness-Script-Hash）** 和多签编码
- 数据存储在 **UTXO 集合** - 比特币核心共识机制的一部分
- **所有完整节点**必须存储此数据以验证区块链
- 数据是**共识关键**的，不可修剪
- 建立在 **Counterparty 协议**上（2014 年建立）

**Ordinals/Inscriptions：**
- 将数据存储在交易的**见证段**
- 见证数据**不是比特币核心共识的一部分**
- 节点可以**修剪见证数据**而不影响共识
- 不**保证**在所有节点上存储
- 依赖**外部索引器**进行发现和验证

### 节点存储保证

| 方面 | Bitcoin Stamps | Ordinals/Inscriptions |
|------|---------------|----------------------|
| **存储位置** | UTXO 集合（共识关键）| 见证数据（非共识）|
| **节点要求** | 所有完整节点存储 | 可以被节点修剪 |
| **共识角色** | 验证所必需 | 共识可选 |
| **永久性** | 只要比特币存在就保证 | 依赖索引器保存 |
| **去中心化** | 真实 - 无外部依赖 | 需要索引器基础设施 |

## 技术架构深度剖析

### Bitcoin Stamps 架构

```
┌─────────────────────────────────────────────────┐
│                  应用层                         │
│           (SRC-20, SRC-721, SRC-101)           │
├─────────────────────────────────────────────────┤
│              Counterparty 协议                  │
│             （资产跟踪和验证）                   │
├─────────────────────────────────────────────────┤
│            P2WSH + 多签编码                     │
│           （嵌入脚本哈希的数据）                 │
├─────────────────────────────────────────────────┤
│             比特币 UTXO 集合                    │
│             （共识关键存储）                    │
├─────────────────────────────────────────────────┤
│              比特币区块链                       │
│            （工作量证明保护）                   │
└─────────────────────────────────────────────────┘
```

### Ordinals/Inscriptions 架构

```
┌─────────────────────────────────────────────────┐
│                  应用层                         │
│              （Ordinal 索引器）                 │
├─────────────────────────────────────────────────┤
│              外部索引器逻辑                     │
│           （不是比特币共识的一部分）             │
├─────────────────────────────────────────────────┤
│              见证数据存储                       │
│             （可以被节点修剪）                  │
├─────────────────────────────────────────────────┤
│              比特币区块链                       │
│            （核心共识忽略内容）                 │
└─────────────────────────────────────────────────┘
```

## 永久性和去中心化

### Bitcoin Stamps 保证

✅ **真正的永久性**：只要比特币存在，数据就存在
✅ **共识关键**：所有节点必须存储和验证
✅ **无外部依赖**：可与任何比特币节点配合使用
✅ **经过验证**：建立在 10+ 年的 Counterparty 协议上
✅ **确定性**：所有节点上相同的数据，相同的结果

### Ordinals/Inscriptions 限制

⚠️ **可修剪数据**：节点可以删除见证数据
⚠️ **索引器依赖**：需要专业软件解释
⚠️ **非共识**：不是比特币验证规则的一部分
⚠️ **碎片化风险**：不同索引器可能有不同数据
⚠️ **发现问题**：没有查找铭文的共识机制

## 技术摘要

Bitcoin Stamps 通过使用已建立的协议将数据存储在比特币的共识关键 UTXO 集合中来实现真正的永久性。Ordinals/Inscriptions 将数据存储在不是比特币共识一部分的可修剪见证段中，需要外部基础设施进行解释。

对于需要最大永久性和去中心化保证的数字资产，Bitcoin Stamps 提供了与比特币实际设计原则一致的技术上优越的架构。
