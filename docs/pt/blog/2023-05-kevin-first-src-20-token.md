# KEVIN torna-se o primeiro token SRC-20

A 3 de maio de 2023, no bloco 788,041, <EntityMention entity="arwyn">Arwyn</EntityMention> implementou <EntityMention entity="kevin" variant="cultural">KEVIN</EntityMention> como o primeiro token <EntityMention entity="src-20">SRC-20</EntityMention> de sempre. Surgiu como o Stamp #18,516, e é o momento em que os tokens fungíveis chegaram ao <EntityMention entity="bitcoin-stamps">Bitcoin Stamps</EntityMention>.

Até esse ponto, os stamps eram peças individuais de arte permanente. O SRC-20 colocou uma pergunta diferente: seria possível construir um padrão de tokens fungíveis sobre a mesma garantia de permanência? <EntityMention entity="kevin" variant="cultural">KEVIN</EntityMention> é a resposta, e continua a funcionar.

## KEVIN antes do token KEVIN

<EntityMention entity="kevin" variant="cultural">KEVIN</EntityMention> apareceu primeiro mais cedo, como o Stamp #4258 no bloco 783,718, a 3 de abril de 2023. Essa era a arte. A implementação do token no bloco 788,041 transformou a personagem no ativo SRC-20 génese, um token de lançamento justo (fair launch) sem pré-venda e sem alocação para insiders.

Essa ordem importa. A comunidade reconheceu <EntityMention entity="kevin" variant="cultural">KEVIN</EntityMention> primeiro como arte, depois como a prova de que o padrão de tokens funcionava. Não foi um lançamento de marketing. Foram construtores a testar um novo padrão com dados reais da cadeia.

## Repor a verdade

Durante algum tempo, alguns rastreadores listaram um token diferente como o primeiro SRC-20. Isso estava errado. Uma correção posterior da CoinGecko confirmou <EntityMention entity="kevin" variant="cultural">KEVIN</EntityMention>, no bloco 788,041, como a verdadeira primeira implementação de um SRC-20, não o STAMP. As alturas de bloco não mentem, e a cadeia coloca <EntityMention entity="kevin" variant="cultural">KEVIN</EntityMention> em primeiro lugar.

## Porque é que isto importou

O SRC-20 deu ao Bitcoin Stamps um modelo de tokens baseado em contas que vive no conjunto UTXO, pelo que os saldos herdam a mesma permanência da arte original. As operações de implementação (deploy), cunhagem (mint) e transferência codificam-se todas no Bitcoin. Sem sidechain, sem ponte (bridge), sem host de confiança.

<EntityMention entity="kevin" variant="cultural">KEVIN</EntityMention> tornou-se a referência para a qual todos os tokens posteriores apontam. Provou o padrão, e deu à comunidade uma mascote com significado: somos todos <EntityMention entity="kevin" variant="cultural">KEVIN</EntityMention>.
